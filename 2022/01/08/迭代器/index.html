迭代器

迭代器是迭代取值的工具，每次的结果都依赖于上一次结果。它是一种不依赖索引的取值方法。

迭代器的定义

  1. 实现了__iter__和__next__方法的类属于迭代器
  2. __iter__: 返回self
  3. __next__: 返回下一个数据，如果没有数据了的话，抛出StopIteration异常。并且其取值不依赖索引。
# define class
class iteration(object):
  def __init__(self):
    self.counter = 0
    
  def __iter__(self):
    return self
  
  def __next__(self):
    self.counter += 1
    if self.counter == 4:
      raise StopIteration()
    return self.counter
  
# instantiation
v1 = iteration()
print(v1.__next__())
print(next(v1))
print(v1.__next__())
print(next(v1))
'''output
1
2
3
--StopIteration--
'''

注：v1.__next__()可用next(v1)代替，两者等价

for循环与迭代器

for循环的本质就是迭代器的循环，下面介绍for循环的原理：

  1. for循环接收一个迭代器。
  2. 首先会调用迭代器内的iter方法返回self
  3. 调用迭代器中的next，并赋值给for中的变量名item
  4. 循环直到迭代器抛出StopIteration异常，for会自动捕获异常并结束循环
for item in iteration():
  print(item)
  
'''output
1
2
3
'''

for可迭代对象：凡是可被for循环的对象都为可迭代对象

可迭代对象

迭代器可以用于for中，其中的特殊的next方法是遍历的核心。但我们又注意到for循环首先会执行iter并返回带有next方法的迭代器self，然后再调用next。所以使用for的关键是有没有iter方法和iter方法返回中有没有next方法。所以一个类中定义了iter方法并返回一个迭代器对象，他也可以用于for中，我们把这种类成为可迭代对象。

# define iteratiable class
class iterable_ob(object):
    def __iter__(self):
        return iteration()
    

# instantiation
v2 = iterable_ob()
for item in v2:
    print(item)

'''output
1
2
3
'''

迭代器可迭代对象：迭代器一定是可迭代对象，但可迭代对象不一定是迭代器

可迭代对象实例：range()

python中有许多函数都为可迭代对象。我们以range()为例，检查其是否为可迭代对象。为此我们要检查range()有无iter以及iter是否返回迭代器。

首先我们可以查看range()中变量方法

判断是否为迭代器或可迭代对象

python中内置两个函数供我们方便的检查一个实例是否为迭代器与可迭代对象。

from collections.abc import Iterable, Iterator

able = iterable_ob()
ation = iteration()

print(isinstance(able, Iterable))
print(isinstance(able, Iterator))

print(isinstance(ation, Iterable))
print(isinstance(ation, Iterator))

'''output
True
False
True
True
'''

优点与缺点

优点：

  1. 不依靠索引
  2. 一次只取一个值，节省内存

缺点：

  1. 只能从前往后取，不能去特定值，不够灵活
  2. 无法得知迭代器长度
